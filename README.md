# 충남한양 인사평가 시스템 (CNHYEX)

중앙화된 360° 인사평가 웹앱입니다. 브라우저는 화면·인증 세션만 담당하고, 인사정보·배정·제출·조정·등급·결과공개 상태는 Supabase에 저장합니다.

## 현재 작업 상태

- 이 작업 트리는 **로컬 구현 및 검증 중**입니다.
- 현재 변경분은 GitHub에 푸시하거나 Vercel/Supabase 운영 환경에 적용하지 않았습니다.
- 운영 반영 전에는 스테이징 데이터베이스에 모든 migration을 파일명 순서대로 적용하고, 역할별 수동 검증을 끝내야 합니다.

## 구성

- 화면: `index.html`
- 서버 API: Vercel Functions (`api/*.js`)
- 인증·데이터·RLS: Supabase Auth + PostgreSQL
- 마이그레이션: `supabase/migrations/*.sql`

## 인사평가 거버넌스

- 상대평가 등급은 본사·영업소·정비사 코호트별로 S 5%, A 20%, B 60%, C 10%, D 5%를 최대잔여 방식으로 배정합니다.
- 동점은 조정 후 점수, 원점수, 직원 ID 순으로 결정합니다.
- 1차 조정은 관리자, 2차 조정 및 전자결재 승인은 임원만 수행합니다.
- 결과는 주기 `마감/보관됨`, 전자결재 승인, 결과 게이트 개방, 결과공개가 모두 충족될 때만 일반 사용자에게 보입니다.
- 일반사용자는 평가 대상자 목록·평가 가이드·내 평가 결과만 사용합니다. 임원은 평가관리의 점수 집계/2차 조정과 평가 마감/과거 이력만 사용합니다. 관리자는 전체 기능을 사용합니다.
- 점수는 5지선다 문항의 가중 환산으로 계산하며, 직군과 평가 관계(내부·교류·리더십)에 맞는 문항만 노출됩니다.

## 로컬 검증

```powershell
node --test api/*.test.js
Get-ChildItem api -Filter *.js | ForEach-Object { node --check $_.FullName }
```

`index.html`은 큰 단일 문서이므로, 배포 전에는 브라우저에서 로그인·권한·평가 제출·조정·결과공개까지 역할별로 확인합니다.

## 비밀정보

`.env`나 실제 SMTP/API 키, Supabase service-role key, 직원 CSV를 저장소에 넣지 않습니다. 필요한 환경변수와 운영 점검은 [SECURITY_SETUP.md](SECURITY_SETUP.md)를 참고하세요.
