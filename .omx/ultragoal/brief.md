# CNHYEX 인사평가 운영체계 2.0

## 목표

현재 단일 HTML + Vercel Functions + Supabase 구조를 유지하면서 다음 7개 기능을 로컬 저장소에 구현하고 검증한다.

1. 본사·영업소·정비사 그룹별 상대평가 등급 비중을 S 5%, A 20%, B 60%, C 10%, D 5%로 정확히 강제한다.
2. 관리자(인사담당자)의 1차 조정 후에만 임원의 2차 조정을 허용한다. 관리자 1명이 조정하면 1차 조정 완료로 본다.
3. 2차 조정 완료 후 웹앱 내부 전자결재 승인을 받아야 결과를 공개한다.
4. 일반사용자·임원·관리자의 화면 및 서버 권한을 역할별로 제한한다.
5. 평가 등급 메일과 비밀번호 재설정 메일의 개별·일괄 발송, 감사로그, 중복 방지를 구현한다.
6. 본사 팀원·본사 팀장/부서장·영업소 직원·정비사별 평가 카테고리를 지원한다.
7. 팀장급 이상은 리더십 평가를 받고, 부서장 간 평가는 교류평가로 분류한다.

## 확정 정책

- 상대평가 인원수 배분은 최대잔여법으로 정수화한다. 합계는 항상 그룹 인원수와 일치한다.
- 동점 정렬은 최종 조정점수 내림차순, 원점수 내림차순, 사용자 ID 오름차순으로 결정한다.
- 그룹 분류는 사용자 프로필의 사원구분/근무지/직급을 서버에서 판정하고, 판정 결과를 평가 주기별로 스냅샷한다.
- 관리자 한 명의 1차 조정 저장으로 해당 피평가자의 1차 조정이 완료된다.
- 임원은 1차 조정 완료자만 2차 조정할 수 있다.
- 전자결재는 평가 주기 단위이며 임원 또는 관리자가 결재 요청, 권한 있는 임원이 최종 승인한다.
- 결과는 전자결재 승인과 결과 공개 상태가 모두 참일 때만 피평가자에게 보인다.
- 비밀번호는 메일에 포함하지 않고 Supabase 비밀번호 재설정 링크만 발송한다.
- AI 코칭은 폐기 상태를 유지한다.
- 브라우저 저장소에는 UI 환경설정만 저장하며 모든 업무 상태는 Supabase가 기준이다.

## 직군별 카테고리

- 본사 팀원: 성과, 협업, 성장, 조화
- 본사 팀장·부서장: 리더십, 팀원 육성, 소통, 전략적 사고
- 영업소 직원: 비상대응, 소통 협력, 솔선수범, 갈등 해소
- 정비사: 역량 개발, 정비 능력, 책임/주인의식, 안전의식

## 역할별 화면

- 일반사용자: 평가 대상자 목록, 평가 가이드, 내 평가 결과 보기
- 임원: 일반사용자 화면 + 평가관리의 점수 집계/2차 조정 + 평가 마감/과거 이력
- 관리자: 전체 화면과 전체 서브탭

## 아키텍처 불변조건

- 기존 Supabase Auth와 RLS 경계를 약화하지 않는다.
- service-role 키는 Vercel 서버 함수에서만 사용한다.
- 기존 마이그레이션을 수정하지 않고 새 마이그레이션을 추가한다.
- 점수·조정·결재·메일 상태는 감사 가능한 이벤트를 남긴다.
- 화면 숨김과 별개로 서버 API에서도 역할을 검증한다.
- 기존 평가 제출, 매칭, 검색/필터, CSV, 조정 취소 기능을 회귀시키지 않는다.

## 완료 기준

- 새 SQL 마이그레이션이 멱등적으로 실행 가능하다.
- 상대평가 배분 합계가 모든 그룹 크기에서 정확히 일치한다.
- 1차 조정 없이 2차 조정 API가 거부된다.
- 전자결재 승인 전 본인 결과 API가 점수·등급을 반환하지 않는다.
- 역할별로 허용되지 않은 탭이 숨겨지고 API 요청도 403으로 거부된다.
- 개별·일괄 메일 발송이 감사로그와 중복 방지 키를 남긴다.
- 직군별 질문만 평가 화면에 표시되고 해당 카테고리로 점수가 집계된다.
- 리더십·교류평가가 배정과 대상자 목록에서 구분된다.
- 인라인 JavaScript 구문, 중복 DOM ID, 비밀정보 패턴 검사가 통과한다.

### Stories

1. 운영 스키마 확장::상대평가, 1·2차 조정, 전자결재, 메일 감사, 직군 카테고리, 평가 관계를 저장하는 멱등적 Supabase 마이그레이션을 추가한다.
2. 상대평가 엔진::본사·영업소·정비사 그룹별로 최대잔여법을 사용해 S/A/B/C/D 등급 비율을 정확히 배분하고 서버 결과 API에 적용한다.
3. 조정·전자결재 워크플로::관리자 1차 조정 완료를 임원 2차 조정의 전제조건으로 만들고 주기 단위 내부 전자결재 승인 전 결과 열람을 차단한다.
4. 역할별 권한::일반사용자·임원·관리자의 상단 탭, 평가관리 서브탭, Vercel API 작업 권한을 확정 정책대로 제한한다.
5. 직군별 질문과 평가 관계::직군별 네 개 카테고리와 질문 타깃을 지원하고 리더십·교류·부서 내 평가를 배정과 목록에 표시한다.
6. 메일 발송::등급 안내와 비밀번호 재설정 메일을 개별·일괄 발송하고 감사로그·중복 방지·재시도 결과를 제공한다.
7. 통합 검증과 문서화::기존 기능 회귀, 권한 우회, 결과 공개 게이트, 등급 배분, 메일 감사, 스크립트·DOM·비밀정보 검사를 수행하고 운영 문서를 갱신한다.
